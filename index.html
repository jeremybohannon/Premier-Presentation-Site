Project Renaissance



Mission


To continulally integrate the pcd-workflow build in order to promote frequent code contributions and early bug detection.



Project Description


For this project I performed automated acceptance testing with a multitude of technologies including: Bamboo, Maven, Docker, Hadoop, Apache Spark, Oozie, and Scala. With the coordination of these technologies we are able to run mockdata through our build end-to-end and compare the result to a golden data set. If the two data sets match we know our build is solid and deemed a release candidate.

Jenkins



Jenkins is an open source continuous integration tool that was initially used but later determined not to be the most optimal solution in the long end.

Bamboo




Bamboo is a continuous intgration service from Atlassian that was chosen for this project due to its quick deployment time.

Sonatype Nexus

Sonatype Nexus is an artifcat respository application that stores all of our finalized build dependecies.

Git

Git is a code repositry that keeps track of our current code and previous versions in case a revert is ever warranted.



Docker is a tool that wraps your software in a completey independent system complete with all dependencies. This guarantees that the software will run the same no matter its environment.









Cloudera provides a base image to create a Docker container. This allows us to quickly set up a Hadoop environment which is required to run our build.



Hadoop is a software framework that was used to store mock data and run spark jobs on a cluster





Spark is a big data processings framework built for speed. Spark runs all of our data analysis jobs in the project renaissance pipeline








Oozie is a workflow scheduler system that manages our Hadoop and Spark jobs.













A complex directed acyclic graph of Spark jobs are triggered that performs the magical Project Renaissance data analysis.

Scala is a multiparadigm programming language that produces byte code that runs in the Java Virtual Machine. For our purposes we used Scala to find any and all diferences between the processed mock data and the golden data set.
