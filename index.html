Project Renaissance



Mission


To continulally integrate the pcd-workflow build in order to promote frequent code contributions and early bug detection.



Project Description


For this project I performed automated acceptance testing with a multitude of technologies including: Bamboo, Maven, Docker, Hadoop, Apache Spark, Oozie, and Scala.

With the coordination of these technologies we are able to run mockdata through our build end-to-end and compare the result to a golden data set. If the two data sets match we know our build is solid and deemed a release candidate.

Jenkins



Jenkins is an open source continuous integration tool that was initially used but later determined not to be the most optimal solution in the long end.

Bamboo





Bamboo is a continuous intgration service from Atlassian that was chosen for this project due to its quick deployment time.

Sonatype Nexus




Sonatype Nexus is an artifact respository application that stores all of our finalized build dependecies.

Git




Git is a distrubuted version control system that keeps track of our current code and all previous versions.



Docker is a tool that wraps your software in an independent system complete with all dependencies.

This guarantees that the software will run in the same environment no matter where its deployed.









Cloudera is a company that developes Apache Hadoop based software and developed a base image that was used to quickly set up our Hadoop environment.



Hadoop is a framework that processes large sets of data in a distrubuted computing environment.

Hadoop was used to store data and run spark jobs on a cluster.





Apache Spark is a big data processings framework built for speed and sophisticated analytics.

Spark runs all of our data analysis jobs in the project renaissance pipeline.








Oozie is a workflow scheduler system that connects and runs our Spark, Shell, and Sqoop actions.













A complex directed acyclic graph of Oozie actions are triggered that performs the Project Renaissance data analysis.

Scala is a functional programming language that runs in the Java Virtual Machine.

For our purposes we used Scala for writing Spark jobs and to find any and all diferences between the processed data and the golden data set.

































Upon passing we send it off to Sonatype Nexus!




























Upon failure we simply trash the build and try again.
